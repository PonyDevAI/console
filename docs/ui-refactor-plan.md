# UI 改进方案：供应商页面 + Agent 模型配置

## 一、问题

### 供应商页面（ProviderPage）
1. 供应商列表是 Card 堆叠，按钮太多（激活、测试连接、拉取模型、编辑、详情、删除 6 个按钮挤在一行）
2. 模型列表藏在"详情"展开里，不够直观
3. 应该用表格展示，更紧凑清晰

### Agent 页面（AgentsPage）
1. 模型配置区域直接内嵌在表格行下方，两个 select + 两个按钮占了一整行，表格很臃肿
2. 应该改为：列表只显示当前模型名，点"配置"按钮弹 Modal 去配置

## 二、供应商页面改造

### 列表改为表格

替换现有的 Card 堆叠布局，改为 DataTable 风格的表格：

```
┌──────────┬─────────────────────────┬────────┬──────────────────────┬──────────────────────┐
│ 名称     │ API 端点                │ 状态   │ 模型                 │ 操作                 │
├──────────┼─────────────────────────┼────────┼──────────────────────┼──────────────────────┤
│ OpenAI   │ https://api.openai.com  │ 已激活 │ gpt-4o, gpt-4.1 等   │ 测试 拉取 编辑 删除  │
│ Anthropic│ https://api.anthropic.. │ 未激活 │ opus, sonnet 等       │ 激活 测试 拉取 编辑 删│
└──────────┴─────────────────────────┴────────┴──────────────────────┴──────────────────────┘
```

表格列：
- 名称：供应商名
- API 端点：font-mono 小字显示
- 状态：StatusBadge（已激活/未激活）
- 模型：ChipRow 显示 models（最多显示 3 个 + "+N"），无模型时显示灰色"未配置"
- 操作：按钮组（激活/测试连接/拉取模型/编辑/删除）

### 编辑 Modal 改造

现有的编辑 Modal 保持不变，已经包含了：
- 名称、API 端点、API 密钥
- 关联应用（AppToggleList）
- 可用模型（tag 输入 + 拉取按钮）

不需要改动。

### 切换模式区域

保持现有的切换模式 Card 不变，放在表格上方。

### 删除的内容

- 删除 Card 堆叠布局
- 删除"详情"展开折叠逻辑（expandedIds 等）
- 删除 ChevronDown/ChevronUp 图标导入

## 三、Agent 页面模型配置改造

### 表格增加"当前模型"列

在现有表格中，在"路径"列后面增加"当前模型"列：

```
┌──────────┬────────┬──────────┬──────────┬──────────┬──────────────────┬──────────────────┐
│ 工具     │ 状态   │ 已安装版本│ 最新版本 │ 路径     │ 当前模型         │ 操作             │
├──────────┼────────┼──────────┼──────────┼──────────┼──────────────────┼──────────────────┤
│ Codex    │ 已安装 │ 0.115.0  │ 0.115.0  │ /opt/... │ gpt-4o (OpenAI)  │ 配置 检测 卸载   │
│ Claude   │ 已安装 │ 2.1.74   │ 2.1.74   │ /usr/... │ opus (Anthropic) │ 配置 检测 卸载   │
│ Gemini   │ 已安装 │ 1.0.0    │ 1.0.0    │ /usr/... │ -                │ 检测 卸载        │
│ OpenCode │ 未安装 │ -        │ -        │ -        │ -                │ 安装             │
└──────────┴────────┴──────────┴──────────┴──────────┴──────────────────┴──────────────────┘
```

"当前模型"列显示规则：
- 已安装 + supports_model_config + 有 assignment → 显示 `模型名 (供应商名)`
- 已安装 + supports_model_config + 无 assignment → 显示灰色 "默认"
- 未安装 或 不支持模型配置 → 显示 "-"

### 操作列增加"配置"按钮

对于已安装且 supports_model_config 的 Agent，在操作列增加"配置"按钮（放在最前面）。

### 删除内嵌的模型配置行

删除现有的 `tool.installed && tool.supports_model_config` 条件下渲染的整个 `<tr>` 行（包含两个 select 和应用/恢复默认按钮）。

### 新增模型配置 Modal

点击"配置"按钮弹出 Modal：

```
┌─────────────────────────────────────────┐
│ 模型配置 — Codex                    [×] │
├─────────────────────────────────────────┤
│                                         │
│ 当前模型                                │
│ ┌─────────────────────────────────────┐ │
│ │ gpt-4o (OpenAI)                     │ │
│ └─────────────────────────────────────┘ │
│                                         │
│ 切换模型                                │
│ 供应商                                  │
│ ┌─────────────────────────────────────┐ │
│ │ [OpenAI ▾]                          │ │
│ └─────────────────────────────────────┘ │
│ 模型                                    │
│ ┌─────────────────────────────────────┐ │
│ │ [gpt-4o ▾]                          │ │
│ └─────────────────────────────────────┘ │
│                                         │
│              [恢复默认]  [应用]          │
└─────────────────────────────────────────┘
```

Modal 内容：
1. 顶部显示当前生效的模型（从 currentModels 读取）
2. 供应商下拉框：列出所有 Provider
3. 模型下拉框：根据选中的供应商，列出该 Provider 的 models
4. footer 两个按钮：
   - "恢复默认"（secondary）→ 调用 removeModelAssignment
   - "应用"（primary）→ 调用 setModelAssignment

## 四、需要修改的文件

### 供应商页面

### Agent 页面
- `apps/web/src/pages/AgentsPage.tsx` — 表格加"当前模型"列 + "配置"按钮，删除内嵌配置行，新增 Modal

### 不需要改的
- 后端 API — 不变
- types.ts — 不变

- adapter — 不变

## 五、实现要点

### ProviderPage 改造要点

1. 删除 `expandedIds` state 和 `toggleExpand` 函数
2. 删除 `ChevronDown`/`ChevronUp` 导入
3. 将 Card 列表替换为 `<table>` 结构（参考 AgentsPage 的表格风格）
4. 模型列用 ChipRow 显示，超过 3 个截断显示 "+N"
5. 操作按钮保持不变（激活/测试/拉取/编辑/删除），只是从 Card 内移到表格行内
6. 切换模式区域保持在表格上方
7. 导入/导出按钮保持在页面顶部

### AgentsPage 改造要点

1. 表头增加"当前模型"列（在"路径"和"操作"之间）
2. 每行增加当前模型显示
3. 操作列：已安装 + supports_model_config 的 Agent 增加"配置"按钮
4. 新增 `modelConfigTool` state（当前打开配置 Modal 的 tool，null 表示关闭）
5. 新增 Modal 组件，内含供应商 select + 模型 select + 应用/恢复默认按钮
6. 删除现有的内嵌模型配置 `<tr>` 行
7. 删除不再需要的 `modelSelections` state（改为 Modal 内部 state）

### 样式一致性

- 表格风格统一使用项目现有的样式：`bg-[var(--bg-accent)]` 表头、`border-[var(--border)]` 边框
- Modal 使用现有的 Modal 组件
- 按钮使用现有的 Button 组件
- 下拉框样式与 AgentsPage 现有的 select 一致
- 所有可点击元素加 cursor-pointer
